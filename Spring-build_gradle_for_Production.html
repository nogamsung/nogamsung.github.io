Spring - build.gradle for Production

Code build.gradle for Dockerization and Production

The famous Byeong-Joon Song became a friend of mine recently, and apparently he uses this code for dockerization and production of Spring server. I looked at the code, and it is astonishing.

1Ô∏è‚É£ Versioning Server

Use Git Revision to use it for patch counting.

build.gradle.kts

plugins {
	id ("org.springframework.boot") version "2.7.1"
	id ("io.spring.dependency-management") version "1.0.11.RELEASE"
	id ("java")
}

val major = 0
val minor = 1
val patch = getRevisionCount()

group = "inu-appcenter"
version = "$major.$minor.$patch"
java.sourceCompatibility = JavaVersion.VERSION_11

/**
 * Get Git Revision Count to Utilize When Counting Versions
 */
fun getRevisionCount(): Int {
	val byteOut = ByteArrayOutputStream()
	project.exec {
		commandLine = "git rev-list --count HEAD".split(" ")
		standardOutput = byteOut
	}
	val output = String(byteOut.toByteArray())

	return Integer.parseInt(output.trim())
}

2Ô∏è‚É£ Docker Image Builder

Next step is to add this code for building docker image.

build.gradle.kts

/**
 * Build Docker Image
 */
val dockerUrl: String? by project
val dockerImageName: String? by project
val dockerUsername: String? by project
val dockerPassword: String? by project
val dockerTag: String? by project

tasks.bootBuildImage {
	imageName = "${dockerImageName ?: project.name}:${dockerTag ?: project.version}"

	print("[bootBuildImage] Image Name: $imageName")

	if (dockerUrl != null) {
		isPublish = true
		docker {
			publishRegistry {
				url = dockerUrl
				username = dockerUsername
				password = dockerPassword
			}
		}
	}
}
