GitHub Actions - CI Pipeline for Spring Boot

What is GitHub Actions

GitHub Actions allows developers to automate various tasks. For example, we can build CI pipelines for Spring Boot project. In this post, I am going to go through the steps to build CI pipeline for Spring Boot.

1Ô∏è‚É£ Setting in GitHub

First go to your project repository and click on Actions in the menu.

2Ô∏è‚É£ Set up a workflow

If you want to start from scratch, click on set up a workflow yourself ->, otherwise, click on the specific configuration you need.

I am going to set up from the scratch for the example.

3Ô∏è‚É£ Workflow Code

For the sake of the example, I am going to use IntelliJ IDEA. To do this, you just need to add the workflow file. In the root of your project folder, /.github/workflows/develop-ci.yaml. I have named the file develop-ci.yaml because I am going to build a CI(Continuous Integration) pipeline, but you can name it whatever you want. Nevertheless, the file has to be in this specific path: /.github/workflows/

The whole workflow code is going to look like this

name: Develop - CI

on:
  push:
    branches: [ "develop" ]
  pull_request:
    branches: [ "develop" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Set up JDK
        uses: actions/setup-java@v3
        with:
          distribution: 'adopt'
          java-version: '11'

      - name: Grant execute permission for Gradlew
        run: chmod +x gradlew

      - name: Build with Gradle
        run: ./gradlew build -x test
        env:
          DB_NAME: ${ { secrets.DB_NAME } }
          DB_PORT: ${ { secrets.DB_PORT } }
          DB_HOST: ${ { secrets.DB_HOST } }
          DB_USERNAME: ${ { secrets.DB_USERNAME } }
          DB_PASSWORD: ${ { secrets.DB_PASSWORD } }

      - name: Build and push Docker image
        uses: mr-smithers-excellent/docker-build-push@v5
        with:
          image: gs97ahn/server
          tags: example-latest
          registry: docker.io
          username: ${ { secrets.DOCKER_USERNAME } }
          password: ${ { secrets.DOCKER_PASSWORD } }

Important ‚ùóÔ∏è

In the example above, I have spacing between the curly brace. However, it should be not have spacing.

Workflow Code Explanation

This workflow will run everytime a push or pull request is triggered in develop branch. It will do specific tasks given in the ubuntu-latest environment. Theses are the specific steps that it will go through:

  • Set up JDK
  • Grant execute permission for Gradlew
  • Build with Gradle
  • Build and push Docker image

Basically, it will set up jdk and test the code with the test codes that you have. Afterwards, it will build it to your docker repository.

4Ô∏è‚É£ Setting up Secrets

The $ in the code is the environment variable that you give when you compile and run the program. The reason for being secret is because there are certain variables that you would want it to be secrets like the example below:

  • port number
  • docker repository username
  • docker repository password
  • database username
  • database passwords

Important ‚ùóÔ∏è

If you do not set up this secrets in your github, you will encounter error because your code will require these secrets everytime the code is compiling.

  • git push

You can find the github actions in work by going to actions menu in your github repository.
